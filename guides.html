DNS Records Explained Simply

Goal: understand the 5 records you’ll actually touch: A, AAAA, CNAME, MX, TXT.

  • A: name → IPv4. home.shecknet.com → 203.0.113.5
  • AAAA: name → IPv6. Same idea as A, but IPv6.
  • CNAME: alias one name to another. www → @
  • MX: which mail server handles your email.
  • TXT: freeform text (SPF/DKIM/verification).

Rule of thumb: Use A/AAAA for real endpoints, CNAME for pointers/aliases.

Docker Security Quick Start

  • Use minimal base images (e.g., alpine) and pin versions.
  • Run as non-root: USER app in Dockerfile.
  • Don’t bake secrets into images; use env or secret managers.
  • Limit container capabilities (--cap-drop ALL, allow what you need).
  • Scan images regularly (Trivy/Grype) and track CVEs by tag.

Starter policy: “No root in containers. No latest tags. Scan before deploy.”
